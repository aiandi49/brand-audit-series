Overview · Section 01

What This File Actually Is

metaearth_outreach_platform.html is the most operationally complete and business-ready file audited in this series. It is a fully functional, single-file sales intelligence dashboard built by MetaEarth Initiative — a Tucson, AZ digital marketing and AI automation agency founded in 2024, operated by Lamar Myers. The platform is not a prototype or a mockup. It is a working internal sales tool: a searchable, filterable directory of 23 Tucson industry categories — from dental offices to funeral homes to law firms — each pre-loaded with fabricated-but-realistic digital audit data, a per-industry modal with 8-parameter gap assessments, a revenue leak calculator, and a cold email template system.

The product thesis is sharp and defensible: most Tucson small businesses are bleeding $6,000–$31,000/month in missed leads due to absent AI chatbots, no online booking, poor mobile performance, and zero after-hours phone coverage. MetaEarth Initiative's sales workflow uses this intelligence to send hyper-personalized cold emails leading with exact dollar leak amounts — positioning their AI phone assistant (starting at $297/month) as the fix.

"This is the rarest find in a single-file audit: the core sales engine is already built and functional. The gap is not imagination — it's infrastructure. Add a real backend, and this becomes a legitimate B2B SaaS platform."

Core Product

B2B sales intelligence + AI automation agency platform. Audits local SMB digital presence, quantifies revenue leakage, and generates personalized outreach — all in service of closing recurring monthly AI phone and chatbot contracts.

Implied User Roles

Sales Rep / Founder (Lamar Myers) — uses the tool to identify targets and generate outreach. Future: Account Manager (track deal pipeline), Client (view their own audit report), Admin (manage industries and data).

Implied User Journey

Browse industries → click card → review per-industry audit in modal → "Generate Outreach Email" → send personalized cold email to business → demo → close $297/month recurring contract.

Business Model

Agency selling recurring AI phone assistant + chatbot + SMS contracts at $297–$997/month per SMB client. This platform is the lead generation and sales qualification engine for that offer.

Technology Audit · Section 03

Violations & Dependency Flags

No Firebase. No Databutton. No Riff. No vendor-locked backend of any kind. The file is architecturally clean. The violations below are structural gaps that prevent this from being a real, multi-user, data-driven SaaS — not platform contamination.

✓ No Firebase. No Databutton. No Riff. No proprietary platform code. One minor external dependency: Cloudflare email obfuscation script (cosmetic, remove in production). Clean baseline.
Violation 01All Industry Data is Hardcoded in a Static JS Array

The entire intelligence engine — 23 industries, 8 audit parameters each, revenue leaks, opportunity estimates, priority scores, gap descriptions — lives in a single INDUSTRIES const in a <script> tag. There is no database, no API, no way to add a 24th industry or update a revenue estimate without editing raw HTML. The "real-time audit" is entirely pre-fabricated.

→ Replace with: PostgreSQL industries table + audit_parameters table + FastAPI CRUD endpoints. Admin UI to add/edit industries and update benchmarks. Real-time data pulled from Google Business Profile API and PageSpeed Insights API.

Violation 02"Generate Outreach Email" Button Fires alert() — Entirely Unbuilt

The most critical conversion action in the entire platform — generating a personalized cold email for a specific business — is stubbed with alert('Generating personalized outreach for ${ind.name}...'). Similarly, "Full Audit Report" fires alert('Opening full audit report...'). These are the two buttons a sales rep would click 50 times a day. Neither does anything.

→ Replace with: A FastAPI /outreach/generate endpoint that accepts industry ID + business name and uses the Anthropic API to draft a personalized email using the industry's audit data. One-click copy to clipboard. One-click push to Gmail or SendGrid.

Violation 03No Real Business Data — Audit Scores Are Fabricated

Every "audit" in this platform is a fictional composite. The dental office score of 28/100, the plumbing mobile speed of 24/100, the funeral home with an expired SSL — none of these are real Tucson businesses. The platform claims to use "live Google data and website analysis" in its hero copy, but delivers entirely hardcoded estimates. A sales rep cannot use this to actually audit any specific business without doing all the research manually.

→ Replace with: Google Places API (real GMB data), PageSpeed Insights API (real mobile scores), and a custom web crawler/checker for SSL, chat widget, and booking system detection. Real business records stored in PostgreSQL per-audit.

Violation 04No User Accounts — Single-User Tool

The platform has no concept of authentication, user sessions, or multi-user access. It functions as a personal reference document. There is no way to track which businesses have been contacted, which outreach emails have been sent, which deals are in the pipeline, or what a second team member is working on.

→ Replace with: FastAPI JWT auth. User table with role (Sales Rep, Admin). Per-rep prospect tracking, outreach log, and deal stage management. Team dashboard for the agency owner to see total pipeline value.

Violation 05No CRM / Prospect Pipeline

The platform identifies 23 industry categories but has no mechanism to track individual businesses within those categories. There is no prospect list, no contact record, no outreach history, no follow-up scheduling, no deal stage, no closed/won tracking. The sales workflow described in the 6-step process timeline (Discover → Audit → Outreach → Close) has no data persistence layer to support it.

→ Replace with: businesses table (name, category, GMB URL, contact email, phone), audits table (per-business scan results), outreach_log table (sent emails, dates, status), deals table (stage, monthly value, close date). Or: integrate with HubSpot/Pipedrive via API.

Violation 06Cloudflare Email Obfuscation — Remove in Production

All email addresses are rendered as Cloudflare-obfuscated spans decoded by /cdn-cgi/scripts/5c5dd728/cloudflare-static/email-decode.min.js. This CDN path is Cloudflare-specific and will break if the file is served from any other host. Contact info (lamar@metaearth.us, (520) 258-2714) is not accessible without this external script.

→ Replace with: Plain text email addresses in the React app, protected by a contact form that sends to a FastAPI /contact endpoint with rate limiting. No CDN dependency for basic contact information.

Violation 07Google Fonts CDN — Privacy + Reliability Risk

Bebas Neue, Syne, and JetBrains Mono are loaded from Google Fonts CDN. In a production SaaS context, this exposes user IP addresses to Google on every page load — a GDPR/CCPA compliance concern. Additionally, CDN failures break the typography entirely.

→ Replace with: Self-hosted font files in /public/fonts/. Use @font-face in global CSS. Font files totaling ~120KB — negligible performance cost for full independence.

Gap Analysis · Section 04

What It Needs

The following functional requirements are entirely absent and must be built from scratch. The frontend shell is excellent — the entire investment goes into backend intelligence, real data pipelines, and CRM workflow:

  • 01
    Real Business Data Ingestion Pipeline. Google Places API integration to pull actual Tucson GMB listings by category. PageSpeed Insights API for real mobile scores. Custom HTTP checker for SSL, chat widget detection, online booking system detection, and after-hours phone coverage. Store results in PostgreSQL businesses and audits tables. Schedule re-audits every 30 days.
  • 02
    Authentication & Role-Based Access. JWT auth (FastAPI). User roles: Admin (Lamar), Sales Rep, Client (business owner viewing their own audit). Bcrypt password hashing. Session refresh tokens. Protected routes in React.
  • 03
    AI-Powered Outreach Email Generator. FastAPI /outreach/generate endpoint: receives business name, industry, audit scores, top gaps → calls Anthropic Claude API → returns a fully personalized cold email body with exact dollar amounts and specific gap callouts. One-click copy. One-click push to Gmail via Google OAuth. The current alert() stub is the platform's most critical missing feature.
  • 04
    CRM Prospect Pipeline. Tables: businesses, audits, outreach_log, deals. Deal stages: Identified → Audited → Outreach Sent → Demo Booked → Proposal Sent → Closed Won / Lost. Pipeline view in React with drag-and-drop stage progression. Monthly revenue under management tracker.
  • 05
    Per-Business Audit Report (Shareable Link). The "Full Audit Report" button should generate a publicly accessible, read-only audit page at /report/:business-id. The page shows the real before/after data for that specific business — their actual Google score, actual mobile performance, actual gaps found. This is the asset that gets linked in the cold email and closes the demo.
  • 06
    Outreach Campaign Tracking. Log every email sent with timestamp, recipient, subject, body, and delivery status (via SendGrid/Mailgun webhook). Track open rates, click rates (on the audit report link), and response status. Per-rep stats: emails sent this week, open rate, reply rate, demos booked.
  • 07
    Industry Intelligence Admin Panel. UI for Lamar to update industry benchmark data (average ticket values, call volumes, miss rates), add new industries, edit gap descriptions, adjust the revenue leak formula parameters. Currently requires direct HTML editing — unacceptable for an operational sales tool.
  • 08
    Client Portal (Post-Sale). Once a business is onboarded as a client, they should be able to log in and see their own dashboard: AI phone call volume, missed calls intercepted, bookings created, leads captured, and ROI vs. monthly fee. This is the retention layer that justifies the recurring contract.
  • 09
    Subscription / Billing. Stripe integration for recurring $297–$997/month client billing. Invoice generation. Failed payment retry logic. Cancellation flow with churn survey. Revenue dashboard showing MRR, churn rate, and net new MRR per month.
  • 10
    Security Hardening. Rate limiting on all API endpoints (slowapi). CORS restriction to known origins. Input sanitization on all form fields. SQL injection prevention via SQLAlchemy ORM. HTTPS enforcement. Audit log for all admin actions.

Critical Insight — The Real AI Product

The platform's hero copy mentions an "AI phone assistant" starting at $297/month — but this product is never described in the file. The backend stack must also integrate with whatever telephony AI service MetaEarth is reselling or building (likely Twilio + GPT or a white-label voice AI platform). The outreach platform is the top-of-funnel. The AI phone assistant is the product being sold. Both need to be built or integrated.

Target Architecture · Section 05

Proposed Stack Structure

The target codebase is a clean monorepo with separated /frontend (React + Vite + TypeScript) and /backend (FastAPI + Poetry) directories. The intelligence layer adds a third module: a background worker for scheduled business audits.

React Frontend

  • pages/ (Dashboard, Industries, Prospects, Pipeline, Reports, Admin, Client Portal)
  • components/ (IndustryCard, AuditModal, EmailPreview, PipelineBoard)
  • features/ (auth, industries, outreach, crm, billing)
  • services/ (api client, google-places, stripe)
  • hooks/ (useAudit, useOutreach, usePipeline)
  • store/ (auth, filters, selected industry)
  • React Router v6 + React Query + Zustand
  • Tailwind CSS (or styled-components)

FastAPI Backend

  • routers/ (auth, industries, businesses, audits, outreach, deals, billing, admin, client)
  • schemas/ (Pydantic models per domain)
  • services/ (audit_engine, email_generator, stripe_service, anthropic_service)
  • repositories/ (SQLAlchemy data access layer)
  • models/ (ORM models)
  • workers/ (Celery + Redis for async audit jobs)
  • core/ (config, security, deps, logging)
  • Alembic migrations + pytest

PostgreSQL Schema

  • users (id, role, email, hash, rep_name)
  • industries (id, name, tag, benchmark_leak, ticket_avg)
  • businesses (id, industry_id, name, gmb_url, email, phone)
  • audits (id, business_id, scores JSON, gaps JSON, created_at)
  • outreach_log (id, business_id, user_id, email_body, status, sent_at)
  • deals (id, business_id, stage, monthly_value, close_date)
  • clients (id, business_id, stripe_sub_id, plan, mrr)
  • client_metrics (client_id, calls_intercepted, bookings, leads, date)

Implement /auth, /industries, /businesses, /audits routers. JWT auth with RBAC (Admin, Sales Rep, Client). Pydantic schemas for all request/response shapes. Connect React frontend to real API — replace the hardcoded INDUSTRIES array with React Query fetches.

Create /audits/run/:business_id endpoint that populates a real audit record. Store results in audits table.

AI Outreach Email Generator (Highest Priority Feature)

Build /outreach/generate FastAPI endpoint. Accept: business name, industry, real audit scores, top 3 gaps, estimated leak. Call Anthropic Claude API with structured prompt. Return personalized email body with specific dollar amounts and gap callouts. Connect to modal "Generate Outreach Email" button — replace alert() with real output.

Step 05

React Frontend Rebuild — Port Design System

Convert the HTML design system into React components: IndustryCard, AuditModal, RevenueTable, BeforeAfter, ProcessTimeline, EmailPreview. CRM Pipeline + Outreach Tracking

Build prospect pipeline board (Kanban-style, drag-and-drop deal stages). Outreach log table per business. Email open/click tracking via SendGrid webhook. Rep performance dashboard: emails sent, open rate, reply rate, demos booked, MRR closed. Admin view of total pipeline across all reps.

Step 07

Shareable Audit Report Page

Build public /report/:report_token page — a read-only, beautifully designed audit report for a specific business showing their real scores, before/after visualization, gap list, and a CTA to book a demo with MetaEarth. Step 08

Client Portal + Stripe Billing

Post-sale client login showing: AI call volume, missed calls intercepted, bookings created, leads captured, ROI vs. monthly fee. Stripe integration for recurring billing ($297–$997/month plans). Invoice history. Failed payment retry. MRR dashboard for the admin/founder view.

Step 09

Background Audit Scheduler (Celery + Redis)

Async worker that re-audits all tracked businesses every 30 days. Alert the sales rep if a previously-audited business's score drops (new opportunity) or if a client improves significantly (success story). Scheduled job to scan new GMB listings in Tucson for newly opened businesses.

Step 10

QA, Security & Deployment

pytest coverage ≥ 75% on all API routes. Playwright E2E for outreach generation and client portal flows. OWASP Top 10 review. Rate limiting on all endpoints. Deploy: Railway (FastAPI + Redis + PostgreSQL) + Vercel (React). Configure Sentry for error tracking. Write README with full setup docs.
