SECTION 01 · FILE OVERVIEW

What this file is.

A single-file interactive HTML toolkit for an affiliate marketing program attached to the AI&I Marketplace — a curated collection of 57 AI-powered business systems at $99/month (White Label) and $999 (Full License), with 30% recurring commissions paid to Solana wallets.

Core Product Identity
This is an affiliate recruiter and growth playbook — not a customer-facing marketing site. It exists purely to turn the product owner (and eventually affiliate partners) into a distribution machine. It's a daily operating manual disguised as a web page. It contains copy-ready scripts for 6 platforms, a visual daily schedule, platform-specific best practices, and an interactive checklist. All in a single 926-line HTML file with zero server dependency.
Product
AI&I Marketplace
57 AI business systems. White Label ($99/mo) or Full License ($999). Target: entrepreneurs, side-hustlers, digital product buyers.
User Role
Affiliate Recruiter / Operator
The file is addressed to a single operator running a 0→10 sales sprint. No multi-user assumptions — pure solo playbook.
User Journey
Execute → Track → Iterate
Open file → Read daily schedule → Pull script from relevant tab → Post or send → Check off checklist item. Fully linear, human-executed flow.
Destination URL
aini-marketplace.vercel.app
A separate Vercel-hosted app (not in this file). This kit is a pre-funnel layer — it drives traffic TO that URL, not from it.

SECTION 03 · TECHNOLOGY AUDIT

What breaks the requirements.

Unlike previous files in this series, the AINI Outreach Kit contains no Firebase calls, no proprietary widgets, and no vendor-locked dependencies. The violations here are architectural omissions, not prohibited inclusions — this file was never intended to be an app, but it needs to become one.

⚠ Flag 01
Solana Wallet Payout — No Infrastructure
Every affiliate script and the WarriorForum post explicitly states payouts go to Phantom/Solana wallets with "no threshold, instant access." This is not implemented anywhere. There is no wallet collection form, no payout trigger, no transaction record, and no affiliate ID system. The promise exists in copy but has no backend truth behind it.
FastAPI endpoint for wallet address collection + PostgreSQL affiliate table with wallet_address field + manual or programmatic Solana payout trigger via Phantom SDK or direct RPC.
⚠ Flag 02
Affiliate Link Tracking — Implied But Missing
Scripts direct all traffic to aini-marketplace.vercel.app/affiliate but there is zero affiliate tracking infrastructure. No UTM generation, no referral code system, no conversion attribution. The WarriorForum post mentions "wallet-based tracking (permanent — no cookie expiry)" — but this mechanism does not exist in any code visible in this file or implied in the stack.
FastAPI referral code generation endpoint → PostgreSQL referral_clicks and conversions tables → React affiliate dashboard showing clicks, conversions, earnings per code.
⚠ Flag 03
Checklist State — Session-Only, No Persistence
The interactive checklist resets on every page refresh. The file acknowledges this ("Reset tomorrow") — but for a daily habit tool used repeatedly by the same operator, this means zero streak tracking, zero completion history, and no accountability data over time.
PostgreSQL daily_checklist_completions table with date + item_id + user_id → FastAPI PATCH endpoint → React state hydrated from DB on load. Adds streak counter and 30-day history view.
⚠ Flag 04
Vercel Analytics Referenced — No Dashboard in Kit
The daily schedule explicitly instructs: "Check Vercel Analytics — see which source is driving traffic to your affiliate page." This requires a separate Vercel account and links to zero data inside this tool. The outreach kit doesn't aggregate its own performance data.
Replace Vercel Analytics with a self-hosted analytics pipeline: FastAPI event tracking endpoint → PostgreSQL events table → React analytics tab in the kit showing traffic source, conversion rate per channel, best-performing script by click-through.
⚠ Flag 05
No Affiliate Enrollment Flow
Multiple scripts reference "free to join, no approval required" and direct traffic to aini-marketplace.vercel.app/affiliate. That page is not in this file. If it doesn't exist or isn't functional, every piece of outreach in this kit leads to a dead end. Enrollment, affiliate ID assignment, and onboarding are entirely unbuilt.
React /affiliate page with enrollment form → FastAPI POST /affiliates → PostgreSQL affiliates table → automated welcome email + unique referral code generation on signup.
⚠ Flag 06
Script Performance — Zero Tracking
30 TikTok hook ideas, 4 LinkedIn posts, 4 Reddit scripts, 4 affiliate DMs — none of them have any tracking instrumentation. The user has no way to know which script is generating the most affiliate sign-ups or sales. The "rotate daily" instruction is operationally sound but strategically blind without data.
Add copy-event logging (POST /events on each Copy button click) → PostgreSQL script_usage table → React analytics view ranking scripts by usage and (if conversion data is piped in) by downstream conversion rate.
SECTION 04 · MISSING FUNCTIONAL REQUIREMENTS

What it needs to become a real app.

The AINI Outreach Kit is a content asset masquerading as an app. Converting it into a production-grade affiliate management platform requires building the full operational layer that the copy promises but none of the code delivers.

1
Affiliate Authentication + Identity System
No login, no user identity, no session management. Every affiliate needs an account with a unique referral code, wallet address, and dashboard. FastAPI JWT auth + PostgreSQL users/affiliates tables with role separation (operator vs affiliate).
2
Referral Code + Link Generation Engine
The product's value proposition is that referral tracking is "permanent" and "wallet-based." Neither mechanism is built. Need: unique code generation on enrollment, link builder UI, click/impression/conversion tracking per code, PostgreSQL referrals schema.
3
Solana Payout Infrastructure
Wallet address collection (Phantom-compatible), payout calculation logic (30% of sale price based on tier), payout scheduling or manual trigger, transaction log, confirmation notifications. This is the core financial promise of the product and it is entirely absent.
4
Affiliate Dashboard
Real-time view of: clicks per referral link, sign-ups attributed, sales converted, earnings (pending + paid), payout history. React frontend consuming FastAPI aggregation endpoints. The "10 affiliates × 10 referrals" math in the scripts needs a live ledger behind it.
5
Operator CRM / Outreach Tracker
The checklist resets daily and tracks nothing. Need a persistent outreach log: who was DM'd, what script was used, when, follow-up date, response status. PostgreSQL outreach_log table + React CRM tab in the kit. Turns the "3 DMs/day" habit into an auditable pipeline.
6
Script Performance Analytics
Copy button events should be logged (which script, what time, what platform tab was active). Link downstream to conversion data. Surface highest-performing scripts in a "Top Performers" view. This transforms the outreach kit from a static reference into a feedback loop.
7
Affiliate Enrollment Page + Onboarding Flow
Every script in the kit leads to aini-marketplace.vercel.app/affiliate — that page needs to be a real React enrollment flow: intro copy → form (name, email, wallet address, niche/audience size) → unique code assignment → welcome email → dashboard redirect.
8
Email Notification System
No notifications anywhere. Affiliates need: welcome email on enrollment, new sale notification, payout confirmation. Operator needs: new affiliate sign-up alert, weekly performance digest. FastAPI background tasks + SMTP or SendGrid integration.
9
Rate Limiting + Abuse Prevention
The affiliate program is explicitly "no approval required." Without rate limiting and input validation on the enrollment form, this is an open invitation for spam sign-ups and fake referral clicks. FastAPI middleware for rate limiting + CAPTCHA on enrollment + click fraud detection logic.
10
Admin Panel for Operator
The operator needs visibility across all affiliates: who joined, their referral activity, pending payouts, top performers, fraud flags. React admin UI consuming FastAPI /admin/* routes. Separate from affiliate-facing dashboard. Role-gated.
SECTION 05 · TARGET ARCHITECTURE

The proposed real stack.

The AINI Outreach Kit should become a two-sided affiliate platform: an operator-facing outreach toolkit (expanded from this file) and an affiliate-facing portal (enrollment, dashboard, payout management). React + FastAPI + PostgreSQL with a Solana payout integration layer.

React Frontend
  • pages/OutreachKit (this file, rebuilt)
  • pages/AffiliateEnroll
  • pages/AffiliateDashboard
  • pages/AdminPanel
  • components/ScriptCard (with event logging)
  • components/ChecklistTracker (persistent)
  • components/OutreachCRM
  • components/PayoutHistory
  • services/api.ts (Axios client)
  • state/auth, state/affiliate
FastAPI Backend
  • routers/auth (JWT, login, register)
  • routers/affiliates (CRUD, enroll)
  • routers/referrals (code gen, click tracking)
  • routers/conversions (sales attribution)
  • routers/payouts (Solana payout trigger)
  • routers/analytics (aggregations)
  • routers/admin (operator endpoints)
  • routers/events (script usage logging)
  • services/email (SMTP/SendGrid)
  • services/solana (payout dispatch)
PostgreSQL Schema
  • users (id, email, role, created_at)
  • affiliates (user_id, wallet_address, referral_code, status)
  • referral_clicks (code, ip, referrer, ts)
  • conversions (affiliate_id, product_id, amount, type)
  • payouts (affiliate_id, amount_sol, status, tx_hash)
  • outreach_log (user_id, platform, script_id, contact, status)
  • checklist_completions (user_id, item_id, date)
  • script_events (script_id, platform, user_id, ts)
Environment + Config
  • DATABASE_URL (PostgreSQL)
  • SECRET_KEY (JWT signing)
  • SOLANA_RPC_URL
  • OPERATOR_WALLET_PRIVATE_KEY
  • SMTP_HOST / SENDGRID_API_KEY
  • FRONTEND_URL (CORS origin)
  • COMMISSION_RATE (default 0.30)
  • Alembic for migrations
Solana Integration Note
The Solana payout layer is the highest-risk technical component. Use solana-py on the FastAPI side for transaction signing, or proxy to a managed service like Helius or QuickNode RPC. Wallets should be collected as base58 public keys, validated on input, and stored in the affiliates table. Never store private keys in the DB — use environment-secured operator wallet.
SECTION 06 · IMPLEMENTATION ROADMAP

Step-by-step build plan.

8 sequential phases to transform the AINI Outreach Kit from a static HTML file into a fully operational affiliate management platform with live tracking, Solana payouts, and a persistent operator CRM.

1
Foundation
Repo + Project Structure
Init monorepo: /frontend (Vite React + TypeScript + Tailwind), /backend (FastAPI + SQLAlchemy + Alembic), /infra (Docker Compose for local PostgreSQL + Redis). Establish shared types contract. README with setup instructions.
2
Database
PostgreSQL Schema + Migrations
Define all 8 tables as SQLAlchemy models. Generate Alembic migration scripts. Seed script with the 57 products as a static catalog table. Validate schema with unit tests against a test database.
3
Auth
Authentication Layer
FastAPI JWT auth: register, login, refresh. Role enum: operator | affiliate. React auth context + protected route wrapper. Operator account seeded via ENV variable — no public registration for operators.
4
Affiliate Core
Enrollment + Referral Code System
React /affiliate enrollment page. FastAPI POST /affiliates: validate email, validate Solana wallet address (base58 regex + checksum), generate unique 8-char referral code, insert to DB, dispatch welcome email. GET /affiliates/:code for link landing page attribution.
5
Tracking
Click + Conversion Attribution Pipeline
FastAPI GET /r/:code — logs click to referral_clicks table, redirects to marketplace. FastAPI POST /conversions — called by marketplace on checkout completion, records sale with affiliate_id, product_id, amount, type (white_label | full_license). Earnings calculated: amount × commission_rate.
6
Payouts
Solana Payout Integration
FastAPI POST /payouts/dispatch (admin-only): aggregates unpaid earnings per affiliate, signs SOL transfer transactions from operator wallet using solana-py, stores tx_hash and status in payouts table. Affiliate dashboard shows pending vs paid with Solscan TX links.
7
Outreach Kit
Rebuild This File as React App
Convert the HTML outreach kit into React components. Add persistent checklist (POST /checklist with date + item_id). Add outreach CRM tab (log DMs with contact name, platform, script used, follow-up date, response). Add copy event logging (POST /events on each Copy click). Script analytics view showing usage counts per script.
8
Polish + Deploy
Admin Panel + QA + Repo Readiness
React /admin: affiliate roster, outreach stats, payout queue, fraud flags. Rate limiting via FastAPI slowapi. Input validation on all forms. End-to-end test: enroll affiliate → click link → log conversion → trigger payout → verify TX. Docker Compose for prod. README with full setup. GitHub Actions CI for lint + test.
